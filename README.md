# Telegram admin bot для управления аккаунтами

⚠️ Важно: проект реализует **админ-бота только для одного владельца** и авторизацию аккаунтов через Telegram API (`Telethon`).

Что реализовано:
- Только один админ (`ADMIN_ID`) имеет доступ.
- Кнопки: `Добавить номер` (`рег` / `не рег`), `Номера`, `Статистика`.
- Авторизация номера через код и поддержка 2FA.
- Ввод кода только reply-ответом на служебное сообщение бота (защита от случайного ввода не в тот поток).
- После входа выполняется `ResetAuthorizations` (выход из остальных сессий, оставление текущей).
- Управление номером: `Выйти с аккаунта`, `Получить код` (последнее сообщение от `777000`).
- Хранение статистики и событий в SQLite.

## Ограничения
1. Интеграция с GUI-клиентами Windows (`Auygram`, официальный Telegram Desktop) в этом репозитории не автоматизируется: нужен отдельный Windows RPA-слой (WinAppDriver/AutoHotkey/pywinauto) и выделенная машина.
2. Сценарий `не рег` с `tempmail.ninja` зависит от внешнего сайта и его API/DOM. Здесь добавлен режим и поток в боте, но автоматизацию почты нужно подключить отдельным модулем.

## Установка

1. Создай Telegram API credentials:
   - `api_id`
   - `api_hash`
   на https://my.telegram.org

2. Установи зависимости:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

3. Настрой окружение:

```bash
cp .env.example .env
```

Заполни `.env`:

```env
BOT_TOKEN=...
ADMIN_ID=8540810366
TELEGRAM_API_ID=...
TELEGRAM_API_HASH=...
DB_PATH=bot.db
SESSIONS_DIR=sessions
```

4. Запуск:

```bash
python main.py
```

## Как пользоваться
- `/start` -> откроется меню.
- `Добавить номер` -> `рег`/`не рег` -> отправь номер `+7999...` -> пришлёт код -> отправь код.
- Если код не пришёл, нажми кнопку `🔁 Отправить код повторно`.
- `Номера` -> выбрать номер -> `Выйти с аккаунта` или `Получить код`.
- `Статистика` -> сводка по номерам и ошибкам.

## Рекомендации по продакшену
- Не храни токен в git, только в `.env`.
- Добавь шифрование сессионных файлов.
